# Lecture4-Memory

### 메모리 사전정의 :
##### 프로그램이 실행하는 동안 데이터와 명령어를 저장하기 위한 주소(address)로 식별되는 저장공간.
# #

### 왜 주소가 중요한가?
```
int n = 404;
printf("%i\n", n);
```
##### 컴퓨터 관점에서는 404를 어디에 저장 하였는가? 입니다.
##### 값(value) = 사물함 안에 들어있는 물건
##### 주소(address) = 물건이 들어있는 사물함의 번호
# #

### C에서 크래시(Crash) (프로그램이 충돌 및 갑자기 죽는 것 ) 가 발생하는 이유
##### 1. 건드리면 안되는 사물함을 열려고 했다
##### 2. 다른(틀린) 사물함의 번호를 들고 가서 열려고 했다

#### 결론: 잘못된 주소를 건드리면 바로 터질 수 있음. 
# #

### 이미지(픽셀)은 왜 바이트와 연결됩니까? 
##### 바이트(byte)는 8비트
##### 이미지는 픽셀(pixel) 이라는 점들의 집합체
##### 각 픽셀은 보통 RGB(빨강/초록/파랑) 값으로 표현되고 각 값이 0~255 입니다.
##### R = 0 ~ 255
##### G = 0 ~ 255
##### B = 0 ~ 255
##### = 이미지 편집은 바이트 값을 변경하는 작업.
# #

### 16진수(hex)는 왜 필요한가?

##### 0과1 (2진수)로 주소나 바이트를 쓰면 너무 길어짐 = 비효율적
##### 16진수는 2진수를 짧게 요약해서 보여줌.

##### 2진수 = 쌀 알갱이를 하나하나 세는 느낌
##### 16진수 = 쌀 알갱이를 묶어서 세는 느낌( 더 빠르고 보기 좋음 )
##### 따라서 메모리 주소는 보통 0x... 형태의 16진수를 사용함.
